# マルチユーザー版 DC Scraping プラットフォーム 設計案 (.NET 10 対応)

DC（確定拠出年金）のスクレイピングを行い、不特定多数のユーザーが自分のスプレッドシートへ自動記録できる Web アプリケーションの設計案です。

---

## 1. コンセプト

- **安定した基盤**: **.NET 10 / ASP.NET Core** による堅牢なサーバーサイド実装。
- **サーバーレス DB**: **Neon (PostgreSQL)** を活用し、スケーラビリティとコスト効率を両立。
- **リッチな UI**: **Blazor Web App** によるモダンで高機能なダッシュボード。
- **リッチな Web ダッシュボード**: **Blazor Web App** と **MudBlazor** を活用し、インタラクティブなグラフや資産統計を表示するモダンな UI を提供。
- **設定不要**: 初回実行時にスプレッドシートを自動作成し、ヘッダー行をセットアップ。
- **安全な保管**: UFJ ログイン情報を AES-256 で強力に暗号化し、DB に保管。

---

## 2. システム構成案

### アプリケーション基盤
- **Blazor Web App (.NET 10)**: サーバーサイドレンダリングと WebAssembly を組み合わせた高速な Web UI。

### UI フレームワーク
- **MudBlazor**: マテリアルデザインに基づいたリッチなコンポーネントライブラリ。チャート、データテーブル、ダイアログ等を提供。

### 認証・認可
- **ASP.NET Core Identity**: ユーザー認証、認可、セッション管理。

### データベース / ORM
- **Neon (PostgreSQL)**: サーバーレスデータベース。
- **Entity Framework Core (EF Core)**: 高度な ORM 機能とマイグレーション。

### スクレイピング部分
- **Playwright (.NET)**: ブラウザを制御し、結果をスプレッドシートへ書き込み。実行環境はコンテナ（GitHub Actions / Cloud Run）を想定。

---

## 3. データベース設計 (主要テーブル)

### `Profiles` (ユーザープロフィール)
- `Id`: 文字列 (Identity User ID)
- `Email`: メールアドレス
- `SpreadsheetId`: 連携先スプレッドシート ID
- `UpdatedAt`: 更新日時

### `DcCredentials` (暗号化ログイン情報)
- `UserId`: 文字列 (Identity User ID)
- `EncryptedId`: 暗号化済ログイン ID
- `EncryptedPassword`: 暗号化済パスワード
- `Iv`: 初期化ベクトル
- `UpdatedAt`: 更新日時

---

## 4. セキュリティ設計

ユーザーの UFJ パスワードなどを保護するため、以下の対策を講じます：

1.  **アプリケーション層での暗号化**: .NET の `Aes` クラスを用いて AES-256 暗号化を実施。暗号鍵は環境変数 `MASTER_ENCRYPTION_KEY` としてセキュアに管理。
2.  **Identity による保護**: ASP.NET Core Identity の標準機能を用い、セキュアなサインインと認可を担保。
3.  **トークン管理**: Google Sheets へのアクセスは、ユーザーが認証した資格情報をセキュアに管理した上で実行。

---

## 5. 運用フロー (セットアップ時)

1.  ユーザーがアカウントを作成し、サインイン。
2.  Google Sheets 連携の設定（API 認可）。
3.  UFJ サイトの ID・パスワードを入力。
4.  アプリが Google Drive 内に「DC_Pension_Data」シートを作成し、ヘッダーを生成。
5.  **ダッシュボード**: 収集された資産データに基づき、美しいグラフや最新の評価額が Web 上で即座に確認可能。
