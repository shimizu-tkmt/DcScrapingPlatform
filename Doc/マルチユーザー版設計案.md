# マルチユーザー版 DC Scraping プラットフォーム 設計案

DC（確定拠出年金）のスクレイピングを行い、不特定多数のユーザーが自分のスプレッドシートへ自動記録できる Web アプリケーションの設計案です。

---

## 1. コンセプト

- **パスワード管理不要**: Google OAuth 2.0 によるシングルサインオン。
- **設定不要**: 初回実行時にスプレッドシートを自動作成し、ヘッダー行をセットアップ。
- **安全な保管**: UFJ ログイン情報を AES-256 で強力に暗号化し、DB に保管。
- **低コスト運用**: Vercel, Supabase, Google Cloud Run 等の無料・サーバレス枠を活用。

---

## 2. システム構成案

### フロントエンド / API
- **Next.js (Vercel)**: ユーザーインターフェースおよび管理用 API。

### データベース / 認証
- **Supabase (PostgreSQL)**: ユーザープロファイル、暗号化ログイン情報の保管、Google 認証。

### スクレイピング部分
- **Playwright (Google Cloud Run)**: サーバレス環境でブラウザを実行し、結果をスプレッドシートへ書き込み。

---

## 3. データベース設計 (主要テーブル)

### `profiles` (ユーザープロフィール)
- `id`: UUID (Primary Key)
- `email`: Google アカウントメール
- `spreadsheet_id`: 連携先スプレッドシート ID
- `created_at`: 登録日時

### `dc_credentials` (暗号化ログイン情報)
- `user_id`: UUID (Foreign Key)
- `encrypted_id`: 暗号化済ログイン ID
- `encrypted_password`: 暗号化済パスワード
- `iv`: 初期化ベクトル
- `updated_at`: 更新日時

---

## 4. セキュリティ設計

ユーザーの UFJ パスワードなどを保護するため、以下の対策を講じます：

1.  **アプリケーション層での暗号化**: サーバーサイド (Next.js API) で、AES-256 ブロック暗号方式を用いて暗号化。暗号鍵は環境変数 `MASTER_ENCRYPTION_KEY` としてセキュアに管理。
2.  **データベース保護 (RLS)**: Supabase の Row Level Security を有効化し、他人のデータにアクセスできないことをシステムレベルで担保。
3.  **トークン管理**: Google Sheets へのアクセスは、各ユーザーが承諾した OAuth トークンの範囲内に限定。

---

## 5. 運用フロー (セットアップ時)

1.  ユーザーが Google アカウントでログイン。
2.  Google Drive への書き込み権限を承諾。
3.  UFJ サイトの ID・パスワードを入力。
4.  アプリが Google Drive 内に「DC_Pension_Data」シートを作成し、ヘッダーを生成。
5.  以降、定期スケジュールでスクレイピングが自動実行。
