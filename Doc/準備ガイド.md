# DC Scraping Platform 導入・準備ガイド

このドキュメントでは、マルチユーザー版プラットフォームを構築・公開するために必要な外部サービス（Google Cloud, Supabase, Vercel）の設定手順をまとめます。

---

## 1. Google Cloud 設定 (OAuth & Drive/Sheets API)

ユーザーの Google ログインとスプレッドシート操作のために必要です。

1.  **プロジェクトの作成**: [Google Cloud Console](https://console.cloud.google.com/) で新規プロジェクトを作成します。
2.  **API の有効化**: 以下の API を有効にします。
    - Google Sheets API
    - Google Drive API
3.  **OAuth 同意画面の設定**:
    - `User Type`: 外部 (External)
    - `Scopes`: `./auth/spreadsheets` および `./auth/drive.file` を追加。
4.  **認証情報の作成**:
    - `OAuth 2.0 クライアント ID` を作成します。
    - アプリケーションの種類: `ウェブ アプリケーション`
    - 承認済みのリダイレクト URI: 
        - 開発用: `http://localhost:3000/api/auth/callback/google`
        - 本番用: `https://[あなたのドメイン]/api/auth/callback/google`
    - **Client ID** と **Client Secret** をメモしておきます。

---

## 2. Supabase 設定 (Auth & Database)

ユーザー管理とデータ保存のために使用します。

1.  **プロジェクトの作成**: [Supabase](https://supabase.com/) で新規プロジェクトを作成。
2.  **認証 (Auth) 設定**:
    - Providers メニューから `Google` を有効化。
    - Google Cloud で取得した `Client ID` と `Client Secret` を入力。
3.  **データベース設定**:
    - SQL Editor を使用して、[設計案](マルチユーザー版設計案.md) に基づくテーブルを作成します。
    - Row Level Security (RLS) を有効にし、ポリシーを設定します。

---

## 3. Google Cloud Run 設定 (Scraping Worker)

ブラウザ（Playwright）を実行する環境です。

1.  **Artifact Registry**: Docker イメージを保存するためのリポジトリを作成。
2.  **Cloud Run サービス**:
    - 作成したイメージをデプロイ。
    - 認証不要（またはサービス間認証）として公開。
    - メモリ 1GB 以上を推奨（Playwright 動作のため）。

---

## 4. 環境変数の準備 (Vercel)

Vercel にデプロイする際、以下の環境変数を設定する必要があります。

| 変数名 | 説明 |
| :--- | :--- |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase のプロジェクト URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase の匿名キー |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase の管理用キー（サーバーサイド用） |
| `GOOGLE_CLIENT_ID` | Google OAuth クライアント ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth クライアントシークレット |
| `MASTER_ENCRYPTION_KEY` | 32文字のランダムな文字列（UFJ情報の暗号化用） |
| `SCRAPER_WORKER_URL` | Google Cloud Run のエンドポイント |

---

## 5. 全体スケジュール（目安）

1.  **Step 1**: Google Cloud / Supabase の基盤設定（本ドキュメント参照）。
2.  **Step 2**: Next.js による フロントエンド雛形と Google ログイン実装。
3.  **Step 3**: データベース連携と暗号化ロジックの実装。
4.  **Step 4**: Cloud Run へのスクレイピングワーカーのデプロイ。
5.  **Step 5**: 全体結合テストと公開。
