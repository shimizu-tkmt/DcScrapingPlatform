# 詳細設計書 - DcScrapingPlatform

## 1. データベース設計

### 1.1 AspNetUsers (Identity 標準)
ユーザー認証情報を管理。

### 1.2 Profiles
ユーザー固有の設定を管理。
- `UserId`: `string` (FK to AspNetUsers.Id)
- `SpreadsheetId`: `string` (Google Sheets ID)
- `UpdatedAt`: `DateTime`

### 1.3 DcCredentials
暗号化されたログイン情報を管理。
- `UserId`: `string` (FK to AspNetUsers.Id)
- `EncryptedId`: `byte[]` (AES暗号化)
- `EncryptedPassword`: `byte[]` (AES暗号化)
- `Iv`: `byte[]` (初期化ベクトル)
- `UpdatedAt`: `DateTime`

### 1.4 AssetHistories
過去の資産推移データを保持。
- `Id`: `int` (PK)
- `UserId`: `string` (FK)
- `TotalBalance`: `decimal` (合計評価額)
- `ProfitLoss`: `decimal` (評価損益)
- `RecordedAt`: `DateTime` (取得日時)

## 2. アプリケーションアーキテクチャ

### 2.1 レイヤー構成
- **Components (UI)**: Blazor コンポーネント (MudBlazor)
- **Controllers/Endpoints**: データ登録、ステータス確認用 API
- **Services**: 暗号化、スクレイピング実行指令、DB 操作、Google API 連携
- **Data (EF Core)**: `ApplicationDbContext` による DB アクセス

## 3. 主要処理フロー

### 3.1 スクレイピング実行フロー
1. GitHub Actions または定期 Job がトリガー。
2. 対象ユーザーの `DcCredentials` を取得し、`MASTER_ENCRYPTION_KEY` で復号。
3. Playwright を起動し、DC サイトへログイン。
4. 資産合計および個別銘柄情報を抽出。
5. **最新の合計評価額等を `AssetHistories` テーブルに保存。**
6. ユーザーの `SpreadsheetId` に基づき、Google Sheets へ追記（バックアップ用）。

### 3.2 ログイン情報の保存フロー
1. ユーザーが平文で ID/PASS を入力。
2. ランダムな `Iv` を生成。
3. `MASTER_ENCRYPTION_KEY` と `Iv` を用いて平文を `Aes` 暗号化。
4. 暗号化データと `Iv` を `DcCredentials` テーブルへ保存。

## 4. UI 設計 (MudBlazor)

### 4.1 ダッシュボード
- `MudChart`: 資産推移を時系列で表示。
- `MudDataGrid`: 最新の資産内訳（ファンド名、評価額、損益等）をグリッド表示。
- `MudAlert`: スクレイピング失敗時の通知。

### 4.2 設定画面
- `MudTextField`: スプレッドシート ID やログイン情報の入力。
- `MudButton`: 暗号化保存およびテスト実行。
