# 詳細設計書 - DcScrapingPlatform

## 1. システム構成と管理
- **機密情報管理**: 
  - ローカル開発: `.NET User Secrets` (secrets.json)
  - 本番/検証: Azure Key Vault または環境変数 (SECRET_CONNECTION_STRING 等)
  - `appsettings.json` には機密情報を含めず、テンプレートのみを配置。
- **環境分離**: Neon (PostgreSQL) のブランチ機能を利用し、`main`（本番）と `dev`（開発）を分離。

## 2. データベース設計

### 2.1 AspNetUsers (Identity 標準)
ユーザー認証情報を管理。

### 2.2 Profiles
ユーザー固有の設定を管理。
- `UserId`: `string` (FK to AspNetUsers.Id)
- `SpreadsheetId`: `string` (Google Sheets ID)
- `UpdatedAt`: `DateTime`

### 2.3 DcCredentials
暗号化されたログイン情報を管理。
- `UserId`: `string` (FK to AspNetUsers.Id)
- `EncryptedId`: `byte[]` (AES暗号化)
- `EncryptedPassword`: `byte[]` (AES暗号化)
- `Iv`: `byte[]` (初期化ベクトル)
- `UpdatedAt`: `DateTime`

### 2.4 AssetHistories
過去の資産推移データを保持。
- `Id`: `int` (PK)
- `UserId`: `string` (FK)
- `TotalBalance`: `decimal` (合計評価額)
- `ProfitLoss`: `decimal` (評価損益)
- `RecordedAt`: `DateTime` (取得日時)

### 2.5 ScrapingLogs (NEW)
スクレイピングの実行履歴とステータスを管理（画面設計のタスク一覧に対応）。
- `Id`: `int` (PK)
- `UserId`: `string` (FK)
- `Status`: `int` (0: 待機中, 1: 実行中, 2: 完了, 3: エラー)
- `Message`: `string` (エラー詳細など)
- `StartedAt`: `DateTime`
- `FinishedAt`: `DateTime?`

## 3. アプリケーションアーキテクチャ

### 3.1 レイヤー構成
- **DcScrapingPlatform.Client (WASM)**: 
  - ダッシュボードや設定画面などのインタラクティブな UI プレゼンテーション。
- **DcScrapingPlatform (Web Server)**:
  - **Account/Identity (SSR)**: 認証に関連する処理（Cookie 認証）。
  - **Endpoints**: 資産データ・ログ閲覧用の Web API。
  - **Services**: 
    - `ExportService`: Google Sheets API 連携（バッチ実行時に使用）。
- **DcScrapingPlatform.Worker (Batch Worker)**:
  - **Role**: GitHub Actions 上で動作し、Playwright によるスクレイピングを一括実行。
  - **Logic**: 全有効ユーザーを順次処理（または並列処理）し、DB を更新。
- **DcScrapingPlatform.Shared (Class Library)**:
  - **Data**: `ApplicationDbContext`, `Migrations`
  - **Models**: `ApplicationUser`, `AssetHistory`, `ScrapingLog`, `DcCredential`
  - **Services**: `EncryptionService` (AES 暗号化/復号)

## 3. 主要処理フロー

### 3.1 スクレイピング実行フロー (GHA定時実行型)
1. **トリガー**: GitHub Actions の Cron スケジュール（例：毎日深夜 2時）。
2. **GHA Runner**: `DcScrapingPlatform.Worker` をビルド・実行。
3. **Worker**: DB から全有効ユーザーの `DcCredentials` を取得。
4. **Worker**: 各ユーザーに対して Playwright でスクレイピング実行。
5. **Worker**: 完了後、`AssetHistories` および `ScrapingLogs` を更新。
6. **Worker**: Google Sheets へデータを追記。

### 3.2 ログイン情報の保存フロー
1. ユーザーが平文で ID/PASS を入力。
2. ランダムな `Iv` を生成。
3. `MASTER_ENCRYPTION_KEY` と `Iv` を用いて平文を `Aes` 暗号化。
4. 暗号化データと `Iv` を `DcCredentials` テーブルへ保存。

### 3.3 初期セットアップ完了チェックフロー
サインアップ後のユーザーがサービスを正常に利用できる状態にするためのガードロジック。
1. 各リクエスト（または Layout レベル）で、ログイン済みユーザーの `Profile.SpreadsheetId` および `DcCredential` の有無を確認。
2. **バリデーションルール**: 「`SpreadsheetId` が存在する」または「`DcCredential` が存在する」のいずれか一方を満たせば「セットアップ完了」とみなす。
3. 未完了の場合、強制的に `/Account/InitialSetup` へリダイレクトし、完了するまで他のページ（Dashboard, etc.）へのアクセスを制限する。

## 5. UI 詳細設計 (MudBlazor)

### 5.1 トップページ / 認証画面
- `MudText`: ヒーローセクションのキャッチコピー
- `MudButton`: ログイン（Variant.Filled, Color.Primary）、アカウント登録（Variant.Outlined）
- `MudTextField`: 入力項目（バリデーションには `MudForm` と配合）

### 5.2 ダッシュボード
- **サマリーカード**: `MudPaper` + `MudText` で実行数・エラー数・総計を表示。
- **資産推移グラフ**: `MudChart` (ChartType.Line)
- **タスク一覧**: `MudDataGrid` を使用し、以下のカラムを表示。
  - ターゲット（UFJ確定拠出年金）
  - ステータス（`MudChip` で色分け: Success, Error, Warning, Info）
  - 最終更新日
  - アクション（ログ表示）

### 5.3 アカウント・連携設定 (既存)
- `MudTabs`: 基本情報と DC 連携設定をタブで切り替え。
- `MudTextField`: スプレッドシート ID, スクレイピング用ログイン ID, パスワード（InputType.Password）。
- `MudMessageBox`: アカウント削除等の危険操作時の確認ダイアログ。

### 5.4 初期設定画面 (NEW)
- **構成**:
    - `MudStepper` または `MudGrid` を用いたセクション構成。
    - DCサイト連携セクション（ID, Pass）。
    - Google Spreadsheet 連携セクション（ID）。
- **バリデーション**: いずれかのセクションが入力されている場合のみ「保存して開始」ボタンを有効化。
- **UI コンポ―ネント**:
    - `MudAlert`: 「最初に使用を開始するための設定を行ってください」という案内を表示。
    - `MudButton`: 「設定を保存してダッシュボードへ」（Color.Primary）。
