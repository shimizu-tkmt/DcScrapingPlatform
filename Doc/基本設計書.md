# 基本設計書 - DcScrapingPlatform

## 1. システム概要
本システムは、確定拠出年金（DC）の資産情報を自動でスクレイピングし、専用の Web ダッシュボードで可視化・管理できるマルチユーザー向け Web サービスです。補助的に Google スプレッドシートへのエクスポート・記録機能も提供します。

## 2. 目的
- 複雑なログインが必要な DC サイトの資産情報を自動取得し、手動入力の手間を省く。
- 複数のユーザーが安全かつ個別に利用できる Web インターフェースを提供。
- 資産推移をダッシュボードで視覚化し、長期的な資産形成を支援。

## 3. システム構成
### 3.1 技術スタック
- **Web App**: ASP.NET Core 10 (Blazor Web App)
- **UI Framework**: MudBlazor
- **Database**: Neon (Serverless PostgreSQL)
- **ORM**: Entity Framework Core
- **Authentication**: ASP.NET Core Identity
- **Scraping**: Playwright (.NET)

### 3.2 インフラ構成
- **Hosting**: Azure App Service / Render (Free Tier)
- **Database Hosting**: Neon.tech (本番: Main Branch, 開発: Dev Branches)
- **Automation**: GitHub Actions (定時スクレイピング実行)

## 4. 機能一覧
### 4.1 ユーザー管理機能
- ユーザー登録・ログイン・ログアウト
- **初期セットアップ**: サインアップ直後のユーザーに対し、以下のいずれかの設定を必須化。
    - DC サイトログイン情報
    - Google スプレッドシート ID
- プロフィール管理（アカウント情報、パスワード設定）
- ※電話番号等の不要な個人情報は収集しない。

### 4.2 DC 連携設定機能
- DC サイトログイン情報の登録（AES-256 暗号化保存）
- 接続確認テスト

### 4.3 ダッシュボード機能 (Web サービス)
- **資産推移の可視化**: DB に蓄積された過去の資産推移をグラフ表示。
- **資産内訳の一覧**: 保有銘柄ごとの評価額、損益の表示。
- **実行状況の確認**: スクレイピングが待機中、実行中、完了、またはエラーであるかの確認。

### 4.4 自動スクレイピング機能
- 定期実行（Cron）によるデータ取得
- Google Sheets へのデータ追記

## 5. セキュリティ設計
- **データ暗号化**: UFJ サイトのログイン情報は `Aes` クラスを用いてアプリケーション層で暗号化。
- **認可**: ユーザーごとに専用のデータを保護（他ユーザーのデータ参照不可）。
- **セキュア通信**: データベース接続および Web 通信はすべて SSL/TLS で保護。
- **機密情報管理**: 接続文字列などの機密情報は、開発環境では **User Secrets**、本番環境では **環境変数** で管理し、ソースコードへの混入を防止。
