# Azure での構築案とコスト比較

Azure を使用してマルチユーザー版プラットフォームを構築する場合の構成案と、提案済み構成（Vercel/Supabase/GCP）との比較です。

---

## 1. Azure での構成案

| コンポーネント | 採用サービス | 無料枠とコスト |
| :--- | :--- | :--- |
| **Frontend / API** | **Azure Static Web Apps** | **無料プランあり**。独自のドメイン接続も無料。 |
| **Authentication** | **Static Web Apps (認証)** | **無料**。Google 等の外部認証を即座に導入可能。 |
| **Database** | **Azure SQL Database** | **初期 12 ヶ月間無料**（それ以降は最小構成で月額 約700円〜）。 |
| **Worker (Scraping)** | **Azure Container Apps** | **従量課金**。実行時のみ課金。月間一定のリクエストまでは実質無料に近い。 |

---

## 2. Azure vs. 提案構成 (Vercel/Supabase/GCP) 比較

| 比較項目 | 提案構成 (Vercel系) | Azure 構成 |
| :--- | :--- | :--- |
| **永久無料枠** | **非常に広い**。50人程度まで0円。 | フロントエンド以外、数年で有料化の可能性。 |
| **Playwright 親和性** | **高い**。Cloud Run で容易に動作。 | やや複雑。コンテナ化が前提となる。 |
| **設定の容易さ** | **高い**。個別の専門サービスのため簡単。 | **中**。Azure 特有の権限管理が必要。 |
| **企業・業務利用** | 中 | **高**。セキュリティと統合管理に強い。 |

---

## 3. なぜ Azure だと少し「難しい」と言われるか

1.  **ブラウザの実行制限**: 
    - Azure Functions（サーバレス）では Playwright に必要なブラウザ（Chromium 等）の依存ライブラリが標準で入っていないため、構築がやや煩雑（Docker コンテナでの構築が必須）になります。
2.  **データベースの無料枠**:
    - Supabase は永久無料枠がありますが、Azure SQL は「12ヶ月無料」が主であり、長期的な「完全無料」での維持が少し難しいです。

---

## 結論

**Azure でも構築は十分可能ですが、個人での「完全無料・手軽さ」を最優先する場合、Vercel + Supabase + GCP の組み合わせが最もコストパフォーマンスに優れています。**

もし、既に Azure の利用経験がある、または会社の方針で Azure に集約したいという場合は、Azure Container Apps + Static Web Apps を使用した構成で詳細設計を書き直すことも可能です。
