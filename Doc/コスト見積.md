# DC Scraping Platform クラウドコスト見積

本プロジェクトで提案している「無料枠を最大限活用した構成」における、ユーザー数ごとのランニングコストの見積もりです。

---

## 1. 採用サービスの無料枠（Free Tier）まとめ

| サービス | 無料枠の内容 |
| :--- | :--- |
| **Vercel** | Hobbyプラン: 月間帯域幅 100GB、独自ドメイン接続無料。 |
| **Supabase** | DB 500MB、認証ユーザー 50,000 MAU、ストレージ 5GB。 |
| **Google Cloud Run** | vCPU秒 180,000、メモリGiB秒 360,000、リクエスト 200万/月。 |
| **GitHub Actions** | 月間 2,000 分までのビルド・実行（プライベートリポジトリ）。 |

---

## 2. ユーザー数別の月額コスト推計（円ベース）

※ 1ユーザーあたり「1日1回スクレイピング実行（約2分/回）」を想定。1ドル=150円で算出。

| ユーザー数 | Vercel | Supabase | Cloud Run | **合計（月額）** |
| :---: | :---: | :---: | :---: | :---: |
| **10人** | 0円 | 0円 | 0円 | **0円** |
| **50人** | 0円 | 0円 | 0円 | **0円** |
| **100人** | 0円 | 0円 | 約 1,000円 | **約 1,000円** |
| **500人** | 0円 | 0円 | 約 7,000円 | **约 7,000円** |

### コスト発生の分岐点
- **Vercel / Supabase**: 個人利用や小規模なコミュニティ運営であれば、ほぼ確実に**永久無料**の範囲内で収まります。
- **Google Cloud Run**: **約50ユーザー**を超えたあたりから、無料枠を使い切るため従量課金が発生し始めます。

---

## 3. 費用を抑えるための工夫

1.  **実行時間の短縮**: スクレイピングのロジックを最適化し、ブラウザの起動時間を 1 分以内に抑えることで、無料枠でカバーできる人数を 2 倍（約100人）まで増やせます。
2.  **実行頻度の調整**: 毎日ではなく「週3回」などに設定を変えることで、コストをさらに半分に圧縮可能です。
3.  **複数プロジェクトの活用**: (上級者向け) Supabase 等の複数プロジェクトを使い分けることで無料枠を増やすことも理論上は可能ですが、管理が複雑になるため推奨はしません。

## 結論

**50名程度の利用規模までは、インフラ費用「完全無料」での運営が可能です。**
それ以上の規模になった場合も、ユーザー1人あたり月額数十円〜100円程度のコスト感でスケールさせることができます。
