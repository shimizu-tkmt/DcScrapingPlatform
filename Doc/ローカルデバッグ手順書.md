# ローカルデバッグ手順書

本ドキュメントでは、開発者がローカル環境で `DcScrapingPlatform` を実行・デバッグするための手順を説明します。

## 1. 前提条件

作業を開始する前に、以下のツールがインストールされていることを確認してください。

- **.NET 10.0 SDK**: [dotnet.microsoft.com](https://dotnet.microsoft.com/download/dotnet/10.0) からインストールしてください。
- **Git**: リポジトリの操作に必要です。
- **PostgreSQL**: ローカルで DB を動かす場合に必要です（Neon を使用する場合は不要）。
- **IDE**: 以下のいずれかを推奨します。
    - **Visual Studio 2022** (Preview 版などの .NET 10 対応版)
    - **VS Code** (C# Dev Kit 拡張機能が必要)

## 2. 環境設定

### 2.1 データベースの準備
本プロジェクトでは PostgreSQL を使用します。
詳細は [SETUP_DB.md](SETUP_DB.md) を参照して、Neon のブランチまたはローカル DB を準備してください。

### 2.2 接続文字列の設定 (User Secrets)
機密情報を `appsettings.json` に記述せず、`.NET User Secrets` を使用して管理します。以下のコマンドをターミナル（PowerShell 等）で実行してください。

```powershell
# プロジェクトディレクトリへ移動
cd Source/DcScrapingPlatform

# 接続文字列を設定 (YOUR_CONNECTION_STRING を実際の値に置き換えてください)
dotnet user-secrets set "ConnectionStrings:DefaultConnection" "YOUR_CONNECTION_STRING"
```

## 3. 実行方法

### コマンドラインからの実行
プロジェクトのルートまたはプロジェクトディレクトリで以下のコマンドを実行します。

```powershell
dotnet run --project Source/DcScrapingPlatform
```
起動後、ブラウザで `https://localhost:7045` または `http://localhost:5000` にアクセスしてください。

## 4. デバッグ方法

### VS Code
1. プロジェクトを VS Code で開きます。
2. `F5` キーを押すか、実行とデバッグビューから「.NET Core Launch (web)」を選択して開始します。
3. ブレークポイントを任意の行に設定してデバッグを行います。

### Visual Studio
1. `Source/DcScrapingPlatform.sln` を開きます。
2. `DcScrapingPlatform` をスタートアッププロジェクトに設定します。
3. `F5` キーを押してデバッグを開始します。

## 5. ホットリロードの利用
コードの変更を即座に反映させるには、以下のコマンドを使用します。

```powershell
dotnet watch --project Source/DcScrapingPlatform
```

---
> [!TIP]
> ブラウザが自動的に開かない場合は、ターミナルに出力される URL を手動で開いてください。
