# 実施記録: GitHub Actions 定時実行へのアーキテクチャ再編（コスト最小化）

## 実施日時
2026-03-18 21:55

## 対象ブランチ
`feature/#41-gha-only-scraping`

## 概要
GitHub Actions (GHA) をスクレイピング実行の唯一の主体とし、Azure Container Apps などの常時/オンデマンド実行環境を廃止することで、運用コストを最小化するアーキテクチャへの再編を行いました。

## 変更内容

### 1. [基本設計書](../../Doc/基本設計書.md)
- **技術スタック**: `Worker App` を `Batch Worker` (GHA Runner) に変更。
- **インフラ構成**: `Azure Container Apps` を削除し、`GitHub Actions` の無料枠を活用する構成に更新。
- **自動スクレイピング機能**: ユーザーによるオンデマンド実行（今すぐ更新）を廃止し、定時実行のみに限定することを明記。

### 2. [詳細設計書](../../Doc/詳細設計書.md)
- **レイヤー構成**: `DcScrapingPlatform.Worker` の役割を「GHA 上で動作するバッチ処理」として再定義。
- **主要処理フロー**: 非同期依頼フローを廃止し、GHA トリガーによる直接的なバッチ実行フローに書き換え。
- **UI 詳細設計**: ダッシュボードのタスク一覧から「即時実行ボタン」を削除。

### 3. [画面設計書](../../Doc/画面設計書.md)
- **ダッシュボード**: タスク一覧のアクションから「即時実行」を削除。
- **連係情報設定**: 旧「初期設定」を統合・整理。

## 検証結果
- 今回は設計ドキュメントの修正のみであるため、各マークダウンファイルが正しく更新され、Git 履歴に整合性が保たれていることを確認。
