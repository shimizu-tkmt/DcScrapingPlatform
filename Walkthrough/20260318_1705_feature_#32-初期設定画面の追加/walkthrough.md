# 初期設定画面の追加と遷移制御の導入 - Walkthrough

## 概要
ユーザーがサインアップした直後や、連携設定（DCサイト、Google Spreadsheet）が未完了の状態でアプリケーションを利用できないようにするため、強制的な初期設定フローを導入しました。

- **プルリクエスト**: [初期設定画面の追加と強制リダイレクトの実装 #36](https://github.com/shimizu-tkmt/DcScrapingPlatform/pull/36)

## 変更内容

### 1. 初期設定画面の作成
- **ファイル**: `InitialSetup.razor` (SSR)
- **機能**:
    - DCサイトのログイン情報（ID/パスワード）と Google Spreadsheet ID の入力を受け付けます。
    - いずれか一方の入力が必須となる相関バリデーションを実装しました。
    - ログイン情報は `IEncryptionService` を使用して安全に暗号化され、データベースに保存されます。

### 2. 強制リダイレクト（ガード）の実装
- **サインアップ後の遷移**: `Register.razor` を修正し、新規登録成功後は直接ダッシュボードではなく `/Account/InitialSetup` へ遷移するようにしました。
- **全画面共通のチェック**: `MainLayout.razor` に、認証済みユーザーがセットアップを完了しているかを API 経由で確認するロジックを追加しました。未完了の場合はどのページにアクセスしても初期設定画面へ引き戻されます。

### 3. 基盤・API の整備
- **状態確認 API**: `SetupController.cs` を新規作成し、現在のユーザーが設定を完了しているかを返すエンドポイントを提供しました。
- **暗号化の改善**: `EncryptionService` に IV（初期化ベクトル）を指定可能なメソッドを追加し、ID とパスワードで IV を共有して保存できるようにしました。

## 検証結果

### 1. ビルド
- 全プロジェクトのビルドが正常に終了することを確認しました。

### 2. 認証・遷移フローの確認
- 未ログイン状態で `/Account/InitialSetup` にアクセスすると、ログイン画面へリダイレクトされることを確認しました。
- 新規登録後、自動的に初期設定画面が表示されることを（コードおよびロジックレベルで）確認しました。
- 初期設定画面で「スプレッドシートID」のみを入力して保存し、その後ダッシュボードへ正常にアクセスできることを確認しました。
